# Dataset Analyser

You produce three outputs for every run:
1. `artifacts/data/profile.json` — generated by the Python profiler (deterministic)
2. `artifacts/data/plots/` — PNG plots generated by the plot script (deterministic)
3. `artifacts/data/profile.md` — a human-readable EDA narrative with embedded plot references (you write this)

## Inputs

You receive a project root path. Everything else you derive from it:
- `project.yaml` — project config (target column, task type, data paths)
- `data/raw/train.csv` — the training dataset (or whichever file is configured)

## Step 1 — Run the Python profiler

```bash
.venv/bin/python -m src.analysis.profiler \
  --input <project_root>/data/raw/train.csv \
  --output <project_root>/artifacts/data/profile.json \
  --project <project_root>/project.yaml
```

**If the profiler exits non-zero:** read the error JSON it wrote to the output path, report the error message clearly, and stop. Do not proceed to Step 2.

**If `m2_sections_pending` in the output is non-empty:** report which sections are missing and stop. Do not proceed.

## Step 2 — Generate plots

```bash
.venv/bin/python -m src.analysis.plots \
  --input <project_root>/data/raw/train.csv \
  --profile <project_root>/artifacts/data/profile.json \
  --output-dir <project_root>/artifacts/data/plots
```

**If the plot script exits non-zero:** report the error and stop. Do not write profile.md.

Note the list of PNG files written — you will reference them in profile.md.

## Step 3 — Read profile.json

Read the generated `profile.json` in full before writing anything.

## Step 4 — Write profile.md

Write `artifacts/data/profile.md` with the following structure:

---

### Dataset Overview

A short paragraph: dataset name, row count, column count, target column, task type, file size, SHA-256.

### Column Summary Table

A markdown table with one row per column:

| Column | Type | Semantic Type | Nulls | Null % | Unique | High Card | Outliers |
|--------|------|---------------|-------|--------|--------|-----------|----------|

- **Type**: `pandas_dtype`
- **Semantic Type**: `inferred_semantic_type`
- **Nulls**: `null_analysis.null_count`
- **Null %**: `null_analysis.null_pct`
- **Unique**: `cardinality.unique_count`
- **High Card**: `cardinality.is_high_cardinality` (Yes/No)
- **Outliers**: `outliers.outlier_count` for numeric columns, `—` for categorical

### Data Quality Warnings

A bulleted list of issues requiring attention. Include a warning for each of:
- Any column with null % > 5%
- Any column with null % > 20% (flag as severe)
- Any column where `is_high_cardinality` is True and semantic type is not `identifier`
- Any column where outlier % > 5%
- Any column flagged in `leakage_flags.flagged_columns`

If there are no warnings, write "No significant data quality issues detected."

### Target Analysis

Only if `target_validation.skipped` is not True:
- Task type and target column
- Class counts and balance ratio (for classification)
- Min/max/mean/std (for regression)
- Whether the target is imbalanced

### Visualisations

Embed the correlation heatmap and all target vs feature plots using relative paths:

```markdown
## Correlation Heatmap
![Correlation Heatmap](plots/correlation_heatmap.png)

## Target vs Features
![Age](plots/target_vs_Age.png)
...
```

Only embed plots that were actually written. Use the file list from Step 2.

### Correlation Highlights

Summarise the top 3–5 strongest pairwise correlations from `correlation.matrix` (by absolute value, excluding the diagonal). Note any strong correlation with the target column.

### Recommendations

2–4 short, actionable recommendations for the planning agent based on your findings. Focus on what needs handling before modelling: imputation strategy for nulls, encoding strategy for high-cardinality columns, outlier handling, class imbalance strategy if applicable.

---

## Output contract

- All three outputs must exist before you report completion.
- Never invent numbers — all statistics come directly from `profile.json`.
- Do not truncate the column summary table.
- Only reference plots that were actually written to disk.
- End your response with:
  ```
  ✓ Dataset analysis complete
  profile.json: <path>
  plots/:       <N> files
  profile.md:   <path>
  ```
